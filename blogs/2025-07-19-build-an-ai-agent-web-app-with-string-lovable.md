<img alt="Build an AI Agent Web App with String + Lovable" src="https://pipedream.com/blog/content/images/2025/07/Lovable---String-2.png" /><p>AI agents are powerful, but they&#x2019;re even more useful when paired with a frontend interface. In this tutorial, we&#x2019;ll walk through how to build a simple web app using <a href="https://www.lovable.so/?ref=pipedream.com" rel="noopener">Lovable</a> that connects to a custom AI research agent built with <a rel="noopener">String</a>. </p><p>The result? A lightweight research assistant that users can access through a clean, minimal web UI. </p><h2 id="what-is-string">What is String?</h2><p><a>String</a> is an AI agent that helps you create event-driven agents using natural language.</p><p>Behind the scenes, it uses <a href="www.pipedream.com">Pipedream&#x2019;s</a> automation engine, giving you access to 2,700+ prebuilt integrations and the ability to write and deploy custom code. You describe what you want your agent to do and String handles the rest.</p><p>Agents built with String can listen for events, interact with web APIs, run custom code, and return structured responses.</p><h2 id="what-is-lovable">What is Lovable?</h2><p><a href="https://www.lovable.so/?ref=pipedream.com" rel="noopener">Lovable</a> is a no-code app builder powered by natural language. It allows anyone to generate fully functional web apps with custom inputs, layouts, and logic  by describing what they want. </p><p>For this project, it serves as the frontend to our String agent, collecting input from users and displaying the agent&#x2019;s output.</p><h2 id="what-we%E2%80%99re-building">What We&#x2019;re Building</h2><p>We&#x2019;ll build a simple web application that allows users to submit the URL of a company they&#x2019;re interested in, along with their email address. The backend agent will then:</p><ul><li>Research the company based on its website</li><li>Summarize what the company does</li><li>Identify its target audience or ICP</li><li>Estimate employee count</li><li>Identify founders or current leadership</li><li>The summarized findings will be sent back to the frontend app and displayed to the user.</li></ul><p>To make it easy to follow along yourself, this was all done within the <strong>free limits</strong> of both apps!</p><h2 id="step-by-step-build-the-agent-and-app">Step-by-Step: Build the Agent and App</h2><h3 id="1-start-with-string">1. Start with String</h3><p>Head over to <a href="www.string.com">String.com</a> and create a new agent using the following prompt:</p><blockquote>I am creating an AI agent where users will submit a website URL via a web form and the agent will then research the company based on the provided URL.<br /><br />The agent needs to research and summarize:<br /><br />- What the company does / what it sells<br />- The company&#x2019;s target audience / ICP<br />- Estimated employee count<br />- Names of the CEO and any co-founders<br /><br />Use the OpenAI Responses API with web search to perform this task.<br /><br />Use an OpenAI Chat API step to consistently format the research as structured JSON and post it back as a custom response to the provided webhook.</blockquote><figure class="kg-card kg-image-card"><img alt="Build an AI Agent Web App with String + Lovable" class="kg-image" height="707" src="https://pipedream.com/blog/content/images/2025/07/String-Agent-Prompt.png" width="969" /></figure><p>This prompt defines the core logic of your agent. Make sure it&apos;s clear that the agent will receive a URL, perform web research, and respond via webhook.</p><p><em>Want to use your own prompt? Read our </em><a href="https://pipedream.com/blog/how-to-write-great-prompts-for-string/"><em>String prompting best practices guide</em></a><em>.</em></p><h3 id="2-copy-the-webhook-url-from-string">2. Copy the Webhook URL from String</h3><p>Once you submit your prompt, String will generate a webhook URL and will begin waiting for an event to use for building the rest of the agent.</p><p>Copy this webhook URL so it can be used when building the web app in Lovable.</p><figure class="kg-card kg-image-card"><img alt="Build an AI Agent Web App with String + Lovable" class="kg-image" height="801" src="https://pipedream.com/blog/content/images/2025/07/String-Webhook-URL.png" width="999" /></figure><h3 id="3-build-the-frontend-with-lovable">3. Build the Frontend with Lovable</h3><p>In a new Lovable project, use this prompt:</p><blockquote>Create a minimalistic UI for a user to submit a prompt to an AI research agent. The prompt will include two values: <br /><br />1. A company URL (just the domain) <br />2. An email address Validate that the email has a valid domain before submission. <br /><br />Post the data to [[INSERT YOUR WEBHOOK URL]]. Once the agent returns a response, format and display the response cleanly in the UI.</blockquote><p>This creates your web app. Lovable will automatically build a form, wire up the webhook submission, and handle response display.</p><figure class="kg-card kg-image-card"><img alt="Build an AI Agent Web App with String + Lovable" class="kg-image" height="850" src="https://pipedream.com/blog/content/images/2025/07/Screenshot_123.png" width="1076" /></figure><h3 id="4-trigger-the-agent-build">4. Trigger the Agent Build</h3><p>Fill in a sample company URL and email in the Lovable app and hit submit. </p><p>This will send an event to your String webhook and allow the agent to start building with real input.</p><h3 id="5-finalize-and-deploy-the-agent">5. Finalize and Deploy the Agent</h3><p>Back in String, the agent should now continue building automatically. It will automatically test the agent, fix any issues, and provide you with the final output once finished.<br /><br />Review the output and make sure it returns the data you expect. If everything looks good, deploy the agent.<br /><br />If you want to make changes, just tell String what you want to adjust then deploy when you are ready.</p><h3 id="6-test-end-to-end">6. Test End-to-End</h3><p>Return to your Lovable app and run another test submission. This time, the deployed agent should receive the request, perform the research, and return the results directly in the web app.</p><p>Once verified, you can publish the app on Lovable and share it with others.</p><figure class="kg-card kg-image-card"><img alt="Build an AI Agent Web App with String + Lovable" class="kg-image" height="975" src="https://pipedream.com/blog/content/images/2025/07/Final-app.png" width="1233" /></figure><h2 id="fine-tuning-your-agent">Fine-Tuning Your Agent</h2><p>String generates solid default prompts, but real-world use often requires iteration. Within the String interface, you can click on the OpenAI step to modify the underlying prompt and instructions. This lets you:</p><ul><li>Adjust tone or output formatting</li><li>Add fallback logic for missing data</li><li>Improve accuracy with more specific guidance</li><li>Prompt engineering is part of the process&#x2014;don&#x2019;t hesitate to tweak and test.</li></ul><figure class="kg-card kg-image-card"><img alt="Build an AI Agent Web App with String + Lovable" class="kg-image" height="1004" src="https://pipedream.com/blog/content/images/2025/07/String-Research-Prompt.png" width="1159" /></figure><h2 id="wrapping-up">Wrapping Up</h2><p>By combining String and Lovable, you&#x2019;ve built a powerful, full-stack AI solution without writing a single line of backend code. The integration of event-driven AI agents and natural language app builders unlocks fast, flexible prototypes that can scale into production-ready tools.</p><p>Try extending this example:</p><ul><li>Add additional fields (e.g., industry or location)</li><li>Allow multiple agents for different research tasks</li><li>Store and email the results to the user</li></ul>
