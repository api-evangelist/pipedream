<img alt="How to Write Great Prompts for String" src="https://pipedream.com/blog/content/images/2025/07/String-Prompt-Box-1.png" /><p><a href="http://string.com/?ref=pipedream.com">String</a> is a powerful tool for building event-driven AI agents, automating workflows, and orchestrating data flows between web APIs. It is especially well-suited for problems that follow a clear &quot;if this, then that&quot; structure.</p><p>Here are examples of the types of prompts <a href="http://string.com/?ref=pipedream.com">String</a> is designed to handle:</p><ul><li>&quot;Build an agent that reviews new Gmail emails, identifies customer requests, drafts a response, and messages me on Slack to review the responses.&quot;</li><li>&quot;When new customers are created in Snowflake, create them as contacts in HubSpot.&quot;</li><li>&quot;Scrape Product Hunt every day at 8pm for the top new products of the day, summarize the launch announcements, and send them to me in an email the following morning at 7am.&quot;</li></ul><p>These are event-based workflows that involve a trigger, one or more actions, and potentially some AI-driven processing along the way.</p><h3 id="what-string-is-not-designed-for">What String is <strong>not</strong> designed for:</h3><ol><li><strong>General questions</strong><br />Example: &quot;What will the weather in San Francisco be like today?</li><li><strong>One-time tasks</strong><br />Example: &quot;What meetings are on my calendar tomorrow?&quot;</li><li><strong>Building front-end applications</strong><br />Example: &quot;Create a website with a home page, pricing page, and a contact form.&quot;</li></ol><p>If your prompt doesn&#x2019;t describe a clear event or an API-based action, String will do its best to help, but the results may be inefficient, incorrect, or impossible to execute.</p><h2 id="why-good-prompts-matter">Why Good Prompts Matter</h2><p>Writing better prompts isn&#x2019;t just about getting better results. It has real downstream impacts on speed, cost, and accuracy.</p><ol><li><strong>Reduce cost: </strong>String uses AI tokens behind the scenes. Poor prompts can waste tokens on unnecessary guesswork or require multiple rebuilds</li><li><strong>Get faster results: </strong>Clear instructions help String build the right automation the first time.</li><li><strong>Avoid invalid outputs: </strong>When users ask for things String isn&#x2019;t designed to do, it still tries to help. This can lead to confusing results or workflows that can&#x2019;t run in production.</li></ol><h2 id="best-practices-for-writing-prompts-in-string">Best Practices for Writing Prompts in String</h2><h3 id="1-always-define-the-trigger">1. Always Define the Trigger</h3><p><strong>Bad:</strong> &#x201c;Build an agent to summarize new emails.&#x201d;<br /><strong>Better:</strong> &#x201c;When I get a new email in Gmail, summarize it with OpenAI and post to Slack.&#x201d;</p><p><strong>Why it matters:</strong> Triggers define how and when the workflow starts. Without one, String will guess or insert a placeholder. This leads to broken flows when you eventually connect to real data.</p><p><strong>Tip:</strong> Triggers can be time-based (e.g. every day at 8 AM), event-based (e.g. new lead in Salesforce), or webhook-based (e.g. form submission - or even a <a href="https://pipedream.com/blog/build-an-ai-agent-with-string-lovable/">custom web app</a>).</p><h3 id="2-be-clear-about-where-the-trigger-comes-from">2. Be Clear About Where the Trigger Comes From</h3><p><strong>Bad:</strong> &#x201c;Start when someone enters info.&#x201d;<br /><strong>Better:</strong> &#x201c;Start when a user submits a form on Tally&#x201d; or &#x201c;Start when a Typeform response is received.&#x201d;</p><p><strong>Why it matters:</strong> String connects APIs. It does not create front-end interfaces or gather user input on its own. If your prompt doesn&#x2019;t specify a data source, String will have nothing to work with and may default to fake or placeholder data.</p><p><strong>Tip:</strong> If you want to capture user input, pair String with a form app such as Tally or Typeform that can send data to a webhook. </p><h3 id="3-name-the-apps-explicitly">3. Name the Apps Explicitly</h3><p><strong>Bad:</strong> &#x201c;Send to my CRM.&#x201d;<br /><strong>Better:</strong> &#x201c;Send to Salesforce&#x201d; or &#x201c;Send to HubSpot.&#x201d;</p><p><strong>Why it matters:</strong> Different apps have different APIs. Being specific helps String route data correctly and prevents errors during execution.</p><p><strong>Tip:</strong> If you aren&#x2019;t sure whether we support your tool, name it anyway. String handles over 2,000 apps and can often find a match.</p><h3 id="4-break-it-down-into-smaller-pieces">4. Break It Down Into Smaller Pieces</h3><p><strong>Bad:</strong> &#x201c;Build an agent that researches companies, enriches them, writes cold emails, and sends them.&#x201d;<br /><strong>Better:</strong></p><p>Step 1: &#x201c;Build an agent that accepts a company website URL submitted through a google form. The agent will then research that company, identifying the Chief Product Officer, VP of Product, Head of Product, or other people in product leadership. &#x201d;</p><p>Step 2: &#x201c;Using the output from the prior agent, draft emails in Gmail for each contact. Send me a Slack message when these are ready to review.&#x201d;</p><p><strong>Why it matters:</strong> Agents often work best when assigned specific tasks. When building a complex, multi-step agentic solution, you should break down your prompt to String by tackling each agentic step one at a time. This will help ensure the first agent works correctly before beginning work on the next agent.</p><p><strong>Tip:</strong> Think like a developer. Break your problem into logical components, then chain them together using multiple agents or workflows.</p><h3 id="5-don%E2%80%99t-over-prompt">5. Don&#x2019;t Over-Prompt</h3><p><strong>Bad:</strong> A giant, 1,000-word prompt copied from ChatGPT that includes every possible instruction.<br /><strong>Better:</strong> A few precise sentences describing the trigger, source app, and desired action.</p><p><strong>Why it matters:</strong> String is not a chatbot or essay writer. It uses LLMs to structure workflows, not to interpret long blocks of natural language. Over-prompting creates confusion and increases cost without improving results.</p><p><strong>Tip:</strong> Start small. Build one part of your workflow at a time, and iterate from there.</p><h2 id="summary-the-anatomy-of-a-great-prompt">Summary: The Anatomy of a Great Prompt</h2><p>If you want String to deliver fast, accurate, and cost-effective results, follow these rules:</p><ol><li>Clearly define what triggers the workflow</li><li>Specify where the triggering the data is coming from</li><li>List the apps String should use to take action</li><li>Break down complex tasks into smaller steps</li><li>Don&apos;t over-prompt</li></ol><p>Think of <a href="http://string.com/?ref=pipedream.com">String</a> like an engineer on your team. The more clearly you define the job, the better the outcome.</p>
