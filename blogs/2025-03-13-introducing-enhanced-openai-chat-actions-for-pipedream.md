<img alt="Introducing Enhanced OpenAI Chat Actions for Pipedream" src="https://pipedream.com/blog/content/images/2025/03/Hero-OpenAI-Responses-API.png" /><p>The Pipedream <a href="https://pipedream.com/apps/openai?ref=pipedream.com">OpenAI app</a> enables a wide range of agentic AI automation use cases by enabling customers to easily add LLM actions powered by OpenAI&#x2019;s models to their workflow.</p><p>OpenAI recently revealed their <a href="https://platform.openai.com/docs/quickstart?api-mode=responses&amp;ref=pipedream.com">Responses API</a> designed to build dynamic AI agent tools. This new API brings built in capabilities for web search, file search, and function calling to the API which adds significant power to what AI agents can do.</p><hr /><h3 id="whats-new">What&apos;s New?</h3><p>The Pipedream <a href="https://pipedream.com/apps/openai?ref=pipedream.com">OpenAI app</a> has been updated to include a new set of actions that use the Responses API:</p><ul><li><a href="https://pipedream.com/apps/openai/actions/chat-using-web-search?ref=pipedream.com">Chat using Web Search</a>: Enhance chat responses with real-time web search results.</li><li><a href="https://pipedream.com/apps/openai/actions/chat-using-file-search?ref=pipedream.com">Chat using File Search</a>: Retrieve and reference data from your own vector store knowledge base, making your chat interactions more informed and context-aware.</li><li><a href="https://pipedream.com/apps/openai/actions/chat-using-functions?ref=pipedream.com">Chat using Functions</a>: Allow your chat model to either answer directly or invoke custom functions - including entire workflows built as functions. This means you can now transform your Pipedream workflows into callable tools, automating complex multi-step tasks seamlessly.</li></ul><h3 id="chat-using-web-search">Chat using Web Search</h3><p>This action lets your chat model integrate live web search results into its responses. Whether you need the latest news, product details, or current data, this action uses OpenAI&#x2019;s advanced web search capabilities to fetch and summarize real-time information.</p><p><strong>Key Props and Configuration</strong></p><ul><li><code>modelId</code>: Choose between <code>gpt-4o</code> or <code>gpt-4o-mini</code> for generating responses.</li><li><code>input</code> &amp; <code>instructions</code>: Provide your chat prompt and any additional context.</li><li><code>previousResponseId</code>: Enable multi-turn conversations.</li><li><code>truncation</code>: Decide whether to automatically truncate responses or disable truncation.</li><li><code>userLocation</code> &amp; <code>searchContextSize</code>: Configure location parameters and context size for more precise search results.</li><li><code>responseFormat</code>: Define whether you want plain text or a structured JSON schema output.</li><li><code>skipThisStep</code>: Pass a boolean to skip executing this step at runtime.</li></ul><h3 id="chat-using-file-search">Chat using File Search</h3><p>Retrieve information from your own file knowledge base stored in vector stores. This is perfect for scenarios where your files contain critical information you want the model to reference when answering questions.</p><p><strong>Key Props and Configuration</strong></p><ul><li><code>modelId</code>: Choose a reasoning model like <code>o3-mini</code> or <code>o1</code>, or use established models like <code>gpt-4o</code>.</li><li><code>input</code> &amp; <code>instructions</code>: Provide your chat prompt and any additional context.</li><li><code>vectorStoreId</code>: The unique identifier of your vector store.</li><li><code>includeSearchResults</code> &amp; <code>maxNumResults</code>: Control how many and whether to include search results from your file store.</li><li><code>metadataFiltering</code>: Enable filtering of search results based on file metadata.</li></ul><h3 id="chat-using-functions">Chat using Functions</h3><p>This action is our most advanced update, leveraging OpenAI&#x2019;s Responses API to allow the chat model to choose between directly answering a query or invoking one of your custom functions. This is where Pipedream truly shines: you can now build entire workflows as functions, which the model can trigger automatically.</p><p><strong>Key Props and Configuration</strong></p><ul><li><code>functions</code>: A JSON array defining functions using OpenAI&apos;s function schema. This is where you list the function names, descriptions, and parameter schemas.</li><li><code>toolChoice</code>: Controls how the model uses functions.</li><li><code>parallelToolCalls</code>: Allow or prevent the model from calling multiple functions in one turn.</li></ul><hr /><h2 id="using-pipedream-workflows-as-functions">Using Pipedream Workflows as Functions</h2><p>One of the most exciting features of this action is the ability to build an entire workflow as a function. Once the model returns a function call, add a custom code step that has all your functions available. Based on the model&apos;s response, the corresponding function or workflow is executed. This empowers you to build robust, multi-step automation directly using a LLM&apos;s power to process natural language.</p><p>The workflow can be as simple as this!</p>

You can provide multiple functions, but for the sake of this guide, we&apos;ll add only one. <a href="https://platform.openai.com/docs/guides/function-calling?api-mode=responses&amp;example=search-knowledge-base&amp;ref=pipedream.com#defining-functions" rel="noopener noreferrer">You can easily generate one by describing what you need here</a>.</p><pre><code class="language-javascript">[
